# 0.4 회원가입 성공 후 닉네임 설정

## 기능 개요

회원가입 완료 후, 사용자가 최초 1회 닉네임을 설정하는 기능

---

## 사용자 흐름

1. 회원가입 완료
2. 닉네임 설정 화면 진입
3. 닉네임 입력
4. "완료" 버튼 클릭
5. 저장 성공 시 → 메인 홈 화면 이동

---

## 입력 조건

| 항목    | 조건            |
| ----- | ------------- |
| 입력값   | 한글, 영문, 숫자 조합 |
| 최대 길이 | 5자            |
| 필수 여부 | 필수 입력         |

---

## 검증 로직

| 케이스    | 처리                     |
| ------ | ---------------------- |
| 미입력    | "닉네임을 입력해주세요."         |
| 형식 불일치 | "한글, 영문, 숫자만 입력 가능해요." |
| 중복 닉네임 | "이미 사용 중인 닉네임이에요."     |

닉네임 입력 시 **중복 체크 API 호출**

---

## 에러 처리

| 상황    | 메시지                     | 방식     |
| ----- | ----------------------- | ------ |
| 저장 실패 | "저장에 실패했어요. 다시 시도해주세요." | 토스트    |
| 입력 오류 | 각 케이스별 메시지              | 인라인 에러 |

---

## 데이터 정책

* 닉네임은 **최초 1회 설정**
* 로그아웃하지 않는 한 **영구 유지**

---

## 추가 고려사항

* "완료" 버튼 비활성화 조건
  → 유효한 닉네임 입력 전까지 disabled
* 중복 체크 API debounce 처리 (UX 개선)
* 최대 글자 수 초과 입력 방지 (입력 자체 제한)
