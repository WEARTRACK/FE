# 닉네임 설정 기능 구현 태스크 플랜

## 1. 목적
- 회원가입 완료 직후 닉네임을 최초 1회 설정하는 플로우를 구현한다.
- 닉네임 입력 검증, 중복 체크(Mock), 저장 처리, 에러 처리까지 사용자 경험을 완성한다.

## 2. 완성조건
- 회원가입 성공 화면 이후 닉네임 설정 화면으로 진입하고, 저장 성공 시 메인홈으로 이동한다.
- 닉네임 규칙(한글/영문/숫자, 2~5자)과 에러 문구가 요구사항대로 동작한다.
- 유효한 닉네임에서만 완료 버튼이 활성화된다.
- 저장 실패 시 iOS 기본 스타일(Alert)로 안내된다.
- 각 작업 완료 후 subagent 리뷰를 받고, 피드백이 코드에 반영된다.

## 3. 해야할 범위
- 엔트리 라우팅/흐름 수정
- 닉네임 설정 화면 UI 및 상태 구현
- 닉네임 중복 체크/저장 Mock API 연동
- 입력 검증/에러 처리/버튼 제어 구현
- 타입체크/린트/핵심 시나리오 검증

## 4. 컨벤션 반영 원칙
- 컴포넌트/파일명은 PascalCase, 변수/함수명은 camelCase를 사용한다.
- 화면/도메인 코드는 `src/features/[domain]` 구조를 따른다.
- 재사용 UI는 `src/components/common`에 둔다.
- 서버 데이터(중복 체크/저장 요청 상태)는 TanStack Query 중심으로 관리한다.
- 전역 앱 상태가 필요한 경우에만 Zustand를 사용한다.
- 머지 전 `npm run lint`, `npm run typecheck`를 반드시 통과한다.

## 5. 작업 태스크
1. 라우팅 및 화면 진입 흐름 구성
- 회원가입 성공 이후 닉네임 설정 화면으로 이동하도록 라우트와 링크를 추가/수정한다.
- 닉네임 저장 성공 시 메인홈으로 이동하도록 내비게이션 동작을 연결한다.
- 완료 기준: 지정 경로에서 닉네임 설정 화면에 진입 가능하고 성공 시 홈 이동이 동작한다.
- 리뷰 게이트: subagent에게 라우팅 회귀/누락 리뷰 요청 후 피드백 반영.

2. 닉네임 설정 화면 UI 구현
- 디자인 시안 기준으로 타이틀, 설명문, 입력창, 글자 수 카운터, 완료 버튼 배치를 구현한다.
- 기본/에러/성공 상태에 맞는 입력 보더 및 메시지 표시를 구현한다.
- 완료 기준: 화면이 시안의 핵심 상태(기본/에러/성공)를 모두 표현한다.
- 리뷰 게이트: subagent UI 리뷰 후 클래스/스타일 피드백 반영.

3. 입력 검증 로직 구현
- 규칙: 필수 입력, 한글/영문/숫자만 허용, 길이 2~5자.
- 오류 우선순위를 정해 인라인 에러 메시지를 일관되게 노출한다.
- 최대 길이 초과 입력은 입력 단계에서 제한한다.
- 완료 기준: 경계값(1,2,5,6자) 및 형식 오류 케이스가 기대 메시지로 동작한다.
- 리뷰 게이트: subagent 검증 로직 리뷰 후 경계 케이스 피드백 반영.

4. 중복 체크(Mock API) 및 debounce 연동
- 입력 안정화 후 중복 체크 Mock API를 호출하는 로직을 구현한다.
- 중복 결과에 따라 에러 메시지와 버튼 활성 상태를 갱신한다.
- 완료 기준: 중복 닉네임은 저장 불가, 사용 가능 닉네임은 저장 가능 상태가 된다.
- 리뷰 게이트: subagent 비동기 상태/경합(race condition) 리뷰 후 피드백 반영.

5. 저장(Mock API) 및 실패 처리 구현
- 완료 버튼 클릭 시 저장 Mock API를 호출한다.
- 성공 시 홈 이동(`replace` 기본값), 실패 시 iOS 기본 스타일 Alert를 노출한다.
- 최초 1회 정책은 백엔드 기준으로 두고, 프론트는 재진입 방지용 보조 처리만 구현한다.
- 완료 기준: 성공/실패 분기 동작과 메시지가 요구사항대로 동작한다.
- 리뷰 게이트: subagent 저장 플로우 리뷰 후 예외 처리 피드백 반영.

6. 통합 검증 및 마무리
- `npm run typecheck`, `npm run lint`를 실행해 정적 검증을 통과한다.
- 핵심 사용자 시나리오(정상/검증 오류/중복/저장 실패)를 수동 점검한다.
- 완료 기준: 타입/린트 오류가 없고 핵심 시나리오가 모두 재현 가능하다.
- 리뷰 게이트: subagent 최종 리뷰 결과 반영 후 종료.

## 6. 결정사항(확정)
- API 전략: Mock 우선
- 플로우: 회원가입 성공 후 닉네임 설정 진입
- 저장 실패 안내: iOS 기본 스타일
- 닉네임 길이: 2~5자
- 최초 1회 정책: 백엔드 기준 + 프론트 보조

## 7. 구현 중 확인 필요 항목(미정 시 기본값 적용)
- Mock 중복 닉네임 샘플 목록: 2~3개 하드코딩 기본
- 저장 성공 후 이동 방식: `replace` 기본
